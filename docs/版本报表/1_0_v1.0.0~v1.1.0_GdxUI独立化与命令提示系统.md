# GdxUI 版本报表 — v1.0.0 ~ v1.1.0

## 一、项目定位

GdxUI 是从 GdxCore 中分离出来的**纯净 UI 组件库**，仅依赖 libGDX + VisUI，不依赖任何引擎框架或业务项目。

**设计原则：**
- 零业务耦合 — 所有控件通过回调/接口与外部交互，不硬编码任何游戏逻辑
- 最小依赖 — 仅 `com.badlogicgames.gdx:gdx` + `com.kotcrab.vis:vis-ui`
- 即插即用 — 一行 `maven { url "https://jitpack.io" }` + 一行依赖即可引用

## 二、v1.0.0 — 项目创建与控件迁移

### 2.1 背景

GdxCore 1.2.0 积累了大量 UI 控件，其中部分仅依赖 libGDX + VisUI，不需要引擎框架支撑。
为降低耦合度并支持控件的独立快速迭代，将这些纯净控件分离为独立项目。

### 2.2 迁移清单（10 个控件）

| # | 控件 | 说明 | 原 GdxCore 路径 |
|---|------|------|----------------|
| 1 | `ConsoleTextField` | 控制台输入栏 — 历史翻阅 + Enter 提交 + 焦点追踪 | `ui/widget/ConsoleTextField.java` |
| 2 | `SelectableLabel` | 可选中文本 — 长按复制 + Ctrl+A/C | `ui/widget/SelectableLabel.java` |
| 3 | `TextFieldPasteMenu` | 输入框长按菜单（复制/剪切/粘贴/全选） | `ui/widget/TextFieldPasteMenu.java` |
| 4 | `HoverFocusScrollPane` | 悬停获取滚动焦点的 ScrollPane | `ui/widget/HoverFocusScrollPane.java` |
| 5 | `SimpleNumPad` | 触屏数字键盘弹出面板 | `ui/widget/SimpleNumPad.java` |
| 6 | `DrawerPanel` | 四方向抽屉面板（带缓动动画） | `ui/widget/DrawerPanel.java` |

## 三、v1.1.0 — ConsoleTextField MC 风格命令提示系统

### 3.1 背景

SandTank 的 ChatOverlay 需要 MC Bedrock 风格的命令提示（输入 `/` 显示建议，Tab 补全），
原先实现在 ChatOverlay 中与 CommandRegistry 硬耦合。
本次将提示机制下沉到 ConsoleTextField，通过 `HintProvider` 接口解耦。

### 3.2 新增 API

```java
// 提示条目
public static class HintEntry {
    public final String command;      // 命令名
    public final String description;  // 描述
}

// 提示提供者接口
public interface HintProvider {
    List<HintEntry> getSuggestions(String prefix);
}

// 设置提示提供者
consoleTextField.setHintProvider(provider);
```

### 3.3 实现细节

| 功能 | 实现 |
|------|------|
| **幽灵文本** | `HintTextField` 内部类继承 `VisTextField`，在 `drawText()` 后以灰色 `(0.5, 0.5, 0.5, 0.5)` 绘制当前最佳匹配的后缀部分 |
| **浮动提示弹窗** | `VisTable hintPopup` 在输入栏上方悬浮，半透明黑色背景 `(0,0,0,0.85)`，列出匹配命令 + 描述 |
| **Tab 循环补全** | 按 Tab 在匹配列表中循环，自动填充命令名到输入栏 |
| **弹窗定位** | 在 `act()` 中每帧根据输入栏 Stage 坐标动态计算弹窗位置 |
| **解耦设计** | ConsoleTextField 不 import CommandRegistry，提示逻辑完全由 HintProvider 注入 |

### 3.4 文件变更

| 文件 | 改动 | 说明 |
|------|------|------|
| `ConsoleTextField.java` | +280 | HintEntry, HintProvider, HintTextField, hintPopup, Tab 补全逻辑 |
| `gradle.properties` | 版本 1.0.0 → 1.1.0 | — |

## 四、通用性审计

所有 10 个控件均通过以下审计标准：

- ✅ 无 `import com.goldsprite.sandtank.*` — 零业务引用
- ✅ 无 `import com.goldsprite.gdengine.log.*` — 不依赖 GdxCore 日志
- ✅ 无 `import com.goldsprite.gdengine.assets.*` — 不依赖 GdxCore 资源工具
- ✅ 所有外部交互通过回调/接口（`setOnSubmit`, `setHintProvider`, `Consumer` 等）
- ✅ 仅依赖 `com.badlogic.gdx.*` + `com.kotcrab.vis.*` + `java.*` 标准库

**结论：GdxUI 是纯净的通用 UI 组件库，无任何业务泄漏。**
